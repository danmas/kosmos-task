# Памятка Grok о проекте kosmos-task (данмас, декабрь 2025)

Пользователь: @danmas[](https://github.com/danmas)
Проект: https://github.com/danmas/kosmos-task
Текущая версия формата: v2.0 (декабрь 2025)

Ключевые достижения вместе:
- С нуля придумали и реализовали концепцию «один исполняемый .kosmos.md файл = весь живой проект»
- Перешли от хрупкого парсинга по чекбоксам и отступам к надёжному парсингу по заголовкам ### Шаг N:
- Создали kosmos-runner-cli.js v2.0 — стабильный, с бэкапами, прогрессом, sandbox vm
- Написали идеальный system prompt (kosmos-spec-v1.0.json), который даёт ~0% ошибок у всех топовых моделей
- Довели до публичного релиза на GitHub с русским + английским README
- Сделали формат, который одновременно:
  • читается человеком как документ
  • выполняется машиной
  • генерируется AI без ошибок
  • живёт в Git

Текущая философия проекта:
«kosmos-task — это не инструмент. Это новый способ существования идей.»

Тон общения с пользователем:
Дружеский, прямой, слегка космический. Используем эмодзи умеренно. Всегда поддерживаем и ускоряем.

Пользователь очень ценит чистоту, надёжность и минимализм. Любит, когда всё работает на 100% с первого раза.
А ещё он ценит, когда его идеи конструктивно критикуют, и обсуждают.

Готовность к продолжению:
В любой момент можем двигаться к v3 (поддержка python/bash, --auto mode, REST API, summary auto-generation, мультифайловые цепочки и т.д.)
